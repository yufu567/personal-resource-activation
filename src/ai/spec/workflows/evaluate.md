# Evaluate Workflow — 价值评估

## 目标
判断资源的激活价值：值不值得投入时间、从哪个角度激活、缺什么东西。

## 输入
- Intake 输出（理解结果 + 领域分类）
- 用户当前活跃目标列表（上下文）
- 历史上该领域的激活成功率（来自 memory/patterns.md）

## 处理步骤

### Step 1: 信号评分
根据资源内在质量打分（0-100），权重：
- 内容密度和可操作性：40%
- 与用户活跃目标的关联度：30%
- 领域知识的新颖程度：20%
- 来源和时效性：10%

### Step 2: 激活方向
- seed (75+)：资源本身有足够信号，可以作为独立激活目标
- fuel (50-74)：资源有补充价值，适合作为其他目标的辅助材料
- archive (<50)：当前信号不足，归档保底，将来可能重新评估

### Step 3: 缺口分析
找出从"资源"到"行动"缺少什么：
- 上下文缺口：资源说了什么但没说为什么
- 用户缺口：用户自己缺少什么背景知识
- 工具缺口：执行需要什么工具或环境
- 验证缺口：怎么判断激活是否成功

## 输出
```json
{
  "valueScore": 78,
  "potential": "seed",
  "recommendation": "建议激活",
  "activationAngles": [
    {"title": "激活角度标题", "how": "怎么从这个角度激活", "confidence": 0.82}
  ],
  "gaps": [
    {"type": "context", "title": "缺口描述", "severity": "high", "howToFill": "怎么补上"}
  ],
  "reasoning": "评分理由"
}
```

## Gate
- valueScore < 30 → 直接建议归档
- 关键缺口未解决 → 暂缓激活
