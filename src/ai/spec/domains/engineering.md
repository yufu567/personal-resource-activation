# 技术/工程领域 (engineering)

## 资源信号
- 包含代码片段、架构图、配置示例
- 讨论技术选型、框架对比、性能优化
- 涉及 API 设计、数据库设计、系统运维
- Git 仓库、技术博客、会议演讲

## 分析视角
1. **可复现性**：是否可以直接复现或需要额外环境？
2. **技术栈匹配**：与用户已有技术栈的关系？
3. **实现成本**：从阅读到可用的时间估算？
4. **时效性**：技术是否已过时？版本是否兼容？

## 激活模式
### 模式 A：代码即资产
资源包含可运行的代码 → 创建本地 demo → 评估复用 → 集成到项目
### 模式 B：知识即参考
资源是概念/方法论 → 提炼核心观点 → 对比现有知识 → 记录差异
### 模式 C：工具即效率
资源介绍新工具 → 试用评估 → 决定是否加入工具链

## 安全注意
- 提示用户注意依赖安全性
- 不直接执行任何代码
- 标注许可证风险
