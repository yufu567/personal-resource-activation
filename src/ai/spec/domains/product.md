# 产品/商业领域 (product)

## 资源信号
- 讨论用户需求、市场分析、竞争格局
- 产品设计文档、PRD、用户故事
- 商业模式分析、增长策略
- 创业经验分享、案例分析

## 分析视角
1. **用户价值**：解决了什么问题？对谁的帮助最大？
2. **可行性**：在当前约束下是否可实现？
3. **差异化**：与现有方案的区别在哪里？
4. **可度量**：如何判断这个产品思路是否有效？

## 激活模式
### 模式 A：策略即方向
资源提供方法论 → 应用到用户自己的产品 → 提出假设 → 设计验证
### 模式 B：案例即参照
资源是别人的成功/失败案例 → 提取关键决策点 → 对比自己的处境 → 列行动清单
### 模式 C：框架即工具
资源提供可复用的框架（如 SWOT、AARRR）→ 填入自己的数据 → 产出分析

## 特殊提醒
- 商业环境差异大，不直接照搬
- 关注时效性：市场数据过期快
