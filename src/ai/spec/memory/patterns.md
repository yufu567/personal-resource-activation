## 通用模式

### 高价值资源特征
- 包含明确、可执行步骤的资源，激活率远高于纯理论描述
- 作者提供了示例代码/案例的资源，更容易转化为行动
- 来自用户主动搜索（而非被动推送）的资源，激活意愿更强

### 低价值资源特征
- 纯标题党、无实质内容 → 激活率接近 0
- 与用户当前目标无关联的随机收藏 → 长期积压
- 内容过时且无历史价值的 → 建议归档

## 领域模式（种子）

### engineering
- 含 running code 的资源 → 激活率最高
- 纯架构图无代码 → 激活率中等，通常作为参考
- 工具介绍类 → 激活取决于用户是否有实际场景

### product
- 有用户数据支撑的案例 → 价值最高
- 纯方法论 → 作为参考框架有价值
- 过时的市场分析 → 归档

### finance
- 提供分析框架 > 提供具体建议
- 数据类资源的时效性敏感，1 个月后价值锐减

### health
- 来自权威医学期刊 > 个人博客
- 任何声称"治愈"的资源自动标记为高风险

### science
- 高引用论文 > 预印本 > 博客解读
- 方法论文比结论论文更有激活价值（方法可复用）

### design
- 有"为什么这样设计"的解释 > 纯视觉展示
- 带有使用场景的设计规范 > 通用设计原则
